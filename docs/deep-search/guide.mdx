# Guide

<p className="subtitle">Models, clients, dashboards, and troubleshooting for Deep Search.</p>

## Overview

Deep Search is an agentic code search tool that uses AI to answer natural language questions about your codebase. Under the hood, it runs an agentic loop that uses Sourcegraph's Code Search and Code Navigation tools to explore your code, gradually refining its understanding until it produces a detailed, sourced answer.

All processing happens within your Sourcegraph instance. The only external calls are made to the configured [LLM](/model-provider).

## Models

Deep Search uses a combination of models selected by Sourcegraph. These are not configurable by end users or site admins — the models are managed by Sourcegraph to ensure optimal performance.

All models are routed through [Cody Gateway](/model-provider). Deep Search is [not available for BYOK customers](/deep-search#custom-model-configuration-and-byok-bring-your-own-key).

## Clients

Deep Search can be accessed through several clients:

### Web — Main page

The primary full-page experience at `/deepsearch`. Users can type a question, use `@`-mentions to scope by repository or file, and view conversation history. Supports the `?q=` query parameter for pre-filling a question.

### Web — Sidebar

Available in the code browsing view (blob and repository pages). Click the Deep Search icon in the sidebar to ask questions with automatic context from the file or repository you are currently viewing.

### Slack

The [Slack integration](/slack-integration) lets users mention `@Sourcegraph` in any channel or thread. Deep Search uses the thread's messages as context. See the [Slack integration page](/slack-integration) for setup and troubleshooting.

### External API

The versioned [Sourcegraph API](/api) at `/api/deepsearch.v1.Service/` provides programmatic access for external clients. Requires access tokens with `externalapi:read` / `externalapi:write` scopes. See the interactive API reference at `/api-reference` on your instance.

The legacy `/.api/deepsearch/v1` REST API is deprecated — see the [migration guide](/deep-search/api).

## Dashboards

Site admins can monitor Deep Search health via a dedicated Grafana dashboard at `/-/debug/grafana/d/deepsearch/deepsearch`. Key panels include:

| Panel | Description |
| --- | --- |
| Questions in flight | Number of questions currently being processed |
| In-flight growth rate | Whether the queue is growing faster than it is being processed |
| Processing rate | Questions processed per minute |
| Error rate | Percentage of questions resulting in errors |
| p99 / p50 duration | Processing time percentiles |
| Fatal LLM stream errors | Errors during LLM streaming that cause failure |
| Non-fatal LLM stream errors | Recoverable LLM streaming errors |

See the full [dashboards reference](/self-hosted/observability/dashboards#deep-search) for details.

## FAQ

### External vs. internal API

| | Internal REST API | External API |
| --- | --- | --- |
| Base URL | `/.api/deepsearch/v1` | `/api/deepsearch.v1.Service/` |
| Protocol | REST / JSON | Connect-RPC (gRPC-compatible) |
| Auth | Any access token | Token with `externalapi:read` / `externalapi:write` scope |
| Intended for | Web UI, internal tools | External integrations, CI/CD, third-party clients |
| Status | Deprecated (legacy) | Supported, versioned |

See the [API migration guide](/deep-search/api) for details on moving from the internal to the external API.

### What is the difference between a source and a citation?

**Sources** are all the artifacts the Deep Search agent *consulted* while answering — files read, searches performed, commits examined, diffs viewed, and repositories explored. They appear in the collapsible "Sources" section of the answer panel.

**Citations** are the specific file references the LLM *linked to* within the answer text. They appear as numbered superscript badges (e.g., `[1]`, `[2]`) inline in the answer, with code snippet previews in the citations panel. Citations are always a subset of sources — specifically, file links with line ranges that the LLM embedded in its markdown answer.

| | Sources | Citations |
| --- | --- | --- |
| What they represent | Everything the agent looked at | What the LLM referenced in the answer |
| Types | File, Search, Commit search, Diff, Repository | File (with line range) only |
| Display | Collapsible "Sources" section | Numbered badges with code previews |

## What to do if...

### Deep Search is not available on my instance

- Verify your plan supports Deep Search ([Enterprise Starter](/pricing/plans/enterprise-starter) or [Enterprise](/pricing/plans/enterprise)).
- Check that `"deepSearch.enabled": true` is set in [site configuration](/deep-search#enabling-deep-search).
- Confirm you are not using BYOK model configuration.
- Ask your site admin to check the Grafana dashboard for errors.

### Questions are stuck in "processing"

- Check the Grafana dashboard at `/-/debug/grafana/d/deepsearch/deepsearch` for in-flight question growth or errors.
- Verify LLM connectivity — Deep Search requires access to Cody Gateway.
- If using the database-backed worker (`deepSearch.worker.enabled: true`), ensure the worker service is running.

### Answers seem incomplete or hit a token limit

- Deep Search will generate a progress summary when the token limit is reached. Use the summary's suggested next steps to start a new, more focused conversation.
- Provide more specific `@`-mentions to narrow the search scope.

### The Slack integration doesn't respond

- See [Slack integration troubleshooting](/slack-integration#troubleshooting).

### API requests return 401 or 403

- For the external API, ensure your access token has `externalapi:read` and/or `externalapi:write` scopes.
- Check that the user has the required RBAC permissions (`DEEP_SEARCH#READ` / `DEEP_SEARCH#WRITE`).
